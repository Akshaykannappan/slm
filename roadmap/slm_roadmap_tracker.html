Implementation Rules

  1. Never copy code from nanoGPT before implementing it yourself.
  2. Implement every module completely from scratch.
  3. Unit test every module independently.
  4. Integrate only after the module passes testing.
  5. Compare your implementation with nanoGPT only after your own version works.
  6. Refactor and optimize after comparison.
  7. Commit each completed module to GitHub with documentation.
  8. Never move to the next phase until the current phase is fully working.
  9. Keep every module independent and reusable.
  10. The final goal is a fully working GPT-style SLM built entirely from first principles.
